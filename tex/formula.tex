\begin{node}
Predicates are actually a little more tricky than I think people want
to give credit, but they're also irrelevant because $FS_{0}$ uses
classes to encode relations.

There are some predicates we could
define, like $A\subset B$ or something similar. In this case, we
should really be using a database to track the predicates, and then
check the sorts of the arguments matches what the database has recorded.
\end{node}

fun subst var tm fm =
  if not(Term.is_var var)
  then raise Fail "\nwlinkedidentc{Formula.subst}{NW1Cmzs6-1jtQRV-1}: first argument must be a variable"
  else if not(Term.are_same_sort var tm)
  then raise Fail "\nwlinkedidentc{Formula.subst}{NW1Cmzs6-1jtQRV-1}: variable and replacement must be same sort"
  else (case fm of
    False => fm
  | (Eq(lhs,rhs)) => Eq(Term.subst var tm lhs, Term.subst var tm rhs)
  | (In(x,cl)) => In(Term.subst var tm x, Term.subst var tm cl)
  | (Pred(P,args)) => Pred(P, map (Term.subst var tm) args)
  | (Not(fm')) => Not(subst var tm fm')
  | (And(A, B)) => And(subst var tm A, subst var tm B)
  | (Or(A, B)) => Or (subst var tm A, subst var tm B)
  | (Implies(A,B)) => Implies(subst var tm A, subst var tm B)
  | (Iff(A,B)) => Iff(subst var tm A, subst var tm B)
  | (Forall(x, fm')) => if Term.eq x var
                        then fm
                        else Forall(x, subst var tm fm')
  | (Exists(x, fm')) => if Term.eq x var
                        then fm
                        else Exists(x, subst var tm fm'));
